VISUAL DNA — NON-NEGOTIABLE
══════════════════════════════════════════════

These elements must appear throughout the site. They define CRAVEE's look:

1. INK DRIP DIVIDERS
   Between every major section, use an SVG ink drip shape as a divider. 
   It should look like thick ink or melted dark liquid dripping downward from 
   the section above. Color: var(--ink) or var(--navy). Never use a straight 
   horizontal line as a section break.

2. FLOATING BUBBLE PARTICLES
   Scattered across the hero and vibe sections — small translucent circles 
   in var(--teal) and var(--ice), animated with CSS keyframes to drift upward 
   slowly at different speeds and sizes (range: 4px to 18px diameter).
   Use at least 20 bubble elements, positioned with absolute/fixed placement.

3. WAVY FLUID BACKGROUNDS
   Section backgrounds should not be flat. Use SVG wave shapes or CSS 
   clip-path: polygon() to create fluid, organic top/bottom edges on sections. 
   Alternate between var(--navy) and var(--navy-mid) across sections.

4. TEAL GLOW EFFECT
   All interactive elements (buttons, cards on hover, active nav items) must 
   have: box-shadow: 0 0 20px rgba(0, 180, 216, 0.4). On hover, intensify to 
   box-shadow: 0 0 40px rgba(0, 180, 216, 0.7).

5. CHARACTER SILHOUETTE
   The cap-wearing character from the logo (the kid eating food, with ink drip 
   body) must appear as a decorative element in at least 2 sections. Recreate 
   it as a CSS-masked div or inline SVG filled with var(--teal) or var(--ink).
   It should feel like a brand mascot haunting the page.

6. AMBER FOOD ACCENTS
   Any food-related element (menu cards, deal cards, food illustrations) must 
   use var(--amber) as an accent — borders, price tags, highlights, icons.
   This creates visual contrast against the blue-dominant palette.

══════════════════════════════════════════════
MOBILE-FIRST — THIS IS CRITICAL
══════════════════════════════════════════════

The primary user is on a phone in Karachi at midnight. Design mobile-first.

Rules:
  - Base styles are for 375px viewport. Scale UP for desktop.
  - Touch targets minimum 48px height on all buttons and nav items.
  - Menu tabs must be horizontally scrollable on mobile (overflow-x: auto, 
    no scrollbar visible: scrollbar-width: none).
  - Hero font scales with clamp(): 
    font-size: clamp(3rem, 12vw, 7rem) for brand name.
  - All cards: full width on mobile, grid on tablet+.
  - Sticky bottom bar on mobile: "📱 ORDER NOW" button always visible, 
    fixed at bottom of screen, full width, teal background, 
    disappears on desktop.
  - Form inputs: font-size minimum 16px to prevent iOS zoom on focus.
  - Three.js hero canvas: disable particle count on mobile (max 30 particles 
    vs 80 on desktop) for performance.
  - Prefers-reduced-motion: wrap all GSAP animations in a check and skip 
    them if the user has reduced motion enabled.
